> 웹 프론트엔드 개발 현장실습<br>
> 키오스크 매출 분석 및 백오피스 웹 서비스 MVP 개발 및 배포

**핵심 성과**
- 프론트엔드 단독 개발로 MVP 버전 완성 및 Netlify 배포
- 전역 상태 캐싱 구조 도입으로 중복 API 호출 제거
- i18n 다국어(한/영) 지원 및 박람회 시연용 인터랙션 UI 구현

## 주요 구현 및 성과

### 매출 분석 UI/UX 개발

- 일간/주간/월간 매출 데이터를 기반으로 하는 **매출 분석 페이지 구조 설계 및 구현**
- API 비동기 요청 처리 및 전역 상태 관리(Redux-store)로 데이터 흐름 일원화

### 성능 개선

- 전역 상태 기반 화면 전환 도입 → **API 호출 횟수 감소**
- 중복된 UI 컴포넌트 리팩토링 → **재사용성 및 유지보수성 향상**

### 기타 개발

- **UI 애니메이션 개발** → Framer-motion 기반 인터랙션
- **글로벌 서비스 기능 개발** → i18n 기반 다국어(한/영) 기능
- **관리자 페이지 기능 개발** → 사진 등 메뉴 정보 업로드 기능
- **배포 및 운영** → Netlify를 활용한 MVP 버전 배포

## 고려 사항

**유지보수성 & 확장성**

- 기능 확장에 유리한 컴포넌트 구조

**기업 활용처 - 박람회 시연**

- 직관적인 API 요청 피드백 화면
- 모달 구성으로 단계적 로직 구성

**UI/UX**

- 전역 상태 및 커스텀 훅을 통한 주간/일간/월간 일관된 상태관리
- 사용성 강화 → 스크롤시 Fade-in 애니메이션 구성

## 이슈 사항

### 중복 API 요청 리팩토링

- **원인:** 일별/주별/월별 화면 전환 시 매번 동일한 API를 재요청하여 중복 호출 발생
- **해결:** 날짜별 및 단위별(일/주/월) 결과를 전역 상태 캐싱, Key 기반 화면 전환 구조로 리팩토링

### 화면 전환 시 사용자 데이터 초기화 문제

- **원인:** 부모 → 자식 → 부모 이동 시 부모 컴포넌트가 재렌더링되며 로컬 상태 초기화
- **해결:** 사용자 데이터를 Redux-store에 전역 상태에 저장하여 상태 유지
